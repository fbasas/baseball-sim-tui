<objective>
Close audit gap 6: DH forfeiture is currently dead code. The SubstitutionManager has the `would_forfeit_dh` method and the `dh_forfeited` flag on SubstitutionRecord, but neither GameEngine.make_substitution() nor the TUI ever sets dh_forfeited=True. This plan wires the detection so a substitution that (a) moves the current DH to a fielding position, or (b) sends a pitcher into a non-pitcher field slot, correctly forfeits the DH for the responsible team.

**Also fixes pre-existing bug:** `src/game/engine.py:229` (and the mirror at line 263 in the position-player branch) uses `isinstance(new_position, type(Position))` — but `type(Position)` is `EnumType` (the metaclass), not `Position`. The check is effectively always False against real Position instances and always True against the Position class itself, which is nonsensical. Since this plan rewrites both branches to derive `old_position` for the new would_forfeit_dh call, the broken derivations at 229-230 and 263 are explicitly DELETED and replaced.

**Note on dependency (per revision NIT):** Plans 01 and 02 modify DISJOINT REGIONS of engine.py (Plan 01: simulate_half_inning + new resolve_pitcher_stats helper; Plan 02: make_substitution). The functions do not call each other and could in principle run in parallel. HOWEVER, both plans modify the same file (src/game/engine.py) and the same test file (tests/test_game_engine.py), so the GSD wave rule "same-wave plans must have zero files_modified overlap" forces Plan 02 to depend on Plan 01 for merge-conflict safety. Wave 2 / depends_on: [06-01] is retained for this reason.

Purpose: Close audit gap 6 (SUBS-03). Plan 03 will then surface this through the TUI's substitution flow.
Output: would_forfeit_dh detects the DH-takes-field case using lineup context; make_substitution stamps dh_forfeited on the record; pre-existing isinstance bug eliminated; targeted tests prove both forfeiture paths and the non-trigger paths.
</objective>
